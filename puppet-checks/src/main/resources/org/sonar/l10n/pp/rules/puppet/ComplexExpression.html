The complexity of an expression is defined by the number of or and and conditions it contains. A single expression's complexity should not become too high to keep the code readable. Try to extract some sub-conditions to make the overall condition more understandable.

This rule checks that the number of conditions is not greater than 3.

Noncompliant Code Example

if $var1 > 2 and $var2 > 3 or ($var3 > $var4 and $var1 < $var3) or $var5 > 4 { }  # Noncompliant: 4 'and' and 'or' conditions

Compliant Solution

$is_position_valid = $var3 > $var4 and $var1 < $var3
if $var1 > 2 and $var2 > 3 or $is_position_valid or $var5 > 4 { }
